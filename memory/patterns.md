# 行为模式库

记录稳定的行为模式，用于指导未来的交互和决策。

---

## 👤 主人偏好模式

### 沟通风格
- **简洁优先**: 主人喜欢直接、简洁的回答
- **选项驱动**: 喜欢看到选项然后选择，而不是被一步步引导
- **系统思维**: 倾向于选择完整方案而非增量改进（见 2026-03-13 反馈）

### 工作习惯
- **时区**: UTC+8，注意时间差异
- **语言**: 中文为主，英文为辅
- **安全意识**: 高（设定了严格的安全红线）

---

## 🛠️ 高效工作流模式

### 任务执行流程
1. **理解**: 确认任务目标
2. **搜索**: 用 memory_search 检索相关历史
3. **执行**: 执行任务
4. **记录**: 记录到 feedback.md
5. **汇报**: 主动向主人汇报结果

### 决策模式
- 不确定时 → 先问主人
- 有选项时 → 列出选项让主人选择
- 外部操作 → 先问再执行（邮件、推文等）

## ✅ 成功模式库

### 长任务处理模式
- **来源**: 2026-03-13 企业微信消息超时问题
- **问题**: 复杂任务执行时间超过 6 分钟，企业微信流式消息过期无法发送
- **模式**: 
  1. **先确认**：收到耗时任务时，先回复"正在处理中，请稍候..."确认收到
  2. **分批执行**：将大任务拆分成小任务，每完成一步汇报进度
  3. **控制时间**：单个任务执行控制在 3 分钟内，超时先汇报进度再继续
- **应用**: 安装依赖、创建多个文件、批量操作等耗时超过 30 秒的任务

### 多能力建设模式
- **来源**: 2026-03-13 五大能力补齐
- **模式**: 主人给出能力清单时，一次性完成所有能力的建设，包括：skill 文件 + 数据文件 + 配置文件 + 依赖安装
- **应用**: 收到多个能力需求时，创建完整的能力栈，而非仅创建文档

### 系统设计模式
- **来源**: 2026-03-13 自我进化系统设计
- **模式**: 主人偏好系统性方案，当有多个改进选项时，倾向于"全部实施"
- **应用**: 涉及系统改进时，主动提供完整方案
